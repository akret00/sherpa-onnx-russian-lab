# Распознавание длинных аудиозаписей (ASR + VAD): бенчмарк моделей

## Цель
- Написать скрипт потокового распознавания длинных аудио без диаризации (ASR + VAD)
- Сравнить 3 модели ASR на записях разной длины и качества
- Метрики: субъективная точность текста, скорость распознавания, потребление ОЗУ, энергоэффективность

> **Характер работы:** это исследовательский эксперимент, первая проба инструментов без глубокого тюнинга параметров. Все модели использовались с конфигурациями «из коробки», предложенными в примерах sherpa-onnx. Результаты отражают baseline-поведение и могут существенно измениться при подборе гиперпараметров (размер окна, пороги VAD, параметры декодирования ASR).

## Алгоритм
- Подпроцесс ffmpeg перекодирует входной файл в 16 кГц, моно, 16-бит PCM и выдаёт поток в stdout
- Инициализация моделей VAD (Silero VAD) и ASR
- Основной цикл:
   - Чтение блока (window_size = 512 сэмплов = 1024 байта = 32 мс)
   - Передача блока в VAD.
   - При обнаружении завершённой фразы — извлечение и распознавание через ASR
   - Вывод текста в консоль
- Завершение подпроцесса ffmpeg

**Исходный код:** [src/asr_vad.py](../src/asr_vad.py)

## Выводы

### Производительность и масштабирование
1. **giga-am-v3 — однозначный выбор для русского языка**:
	- при 1 потоке быстрее qwen3-0.6B в 4.8 раза;
	- при 8 потоках — в 8 раз:
	- потребляет в 5 раз меньше памяти, качество распознавания отличное.
2. **Многопоточность даёт убывающую отдачу по скорости**: прирост 2.1–3.6x при переходе с 1 на 8 потоков вместо теоретических 8x.
3. **Энергопотребление: 8 потоков быстрее и экономичнее одновременно.** Суммарное процессорное время при 8 потоках *растёт* (накладные расходы на синхронизацию), но реальное время падает сильнее, поэтому общая энергия на файл *снижается*:
	   - giga-am, короткий файл: 82 Дж (1 поток) → 51 Дж (8 потоков), экономия **38%**
	   - qwen3-0.6B, короткий файл: 425 Дж → 393 Дж, экономия **8%**
	   - giga-am, длинный файл (1808 с): 1668 Дж → 1175 Дж, экономия **30%**
	   - qwen3-0.6B, длинный файл: 9185 Дж → 8208 Дж, экономия **11%**
   
	Для giga-am эффект выражен сильнее: быстрее и дешевле по энергии одновременно. Для qwen3-0.6B экономия скромная, но тоже есть. Вывод: **8 потоков не «жрут батарею», а наоборот — экономят её** за счёт раннего завершения работы.
### Модели qwen3: нишевое применение
- **Мультиязычность** — главное преимущество перед моноязычной giga-am.
- **0.6B модель требует постобработки**: сырой текст содержит много ошибок, но смысл понятен. Для продакшена нужна вторая LLM для коррекции.
- **1.7B модель непрактична на CPU**: на 1 потоке распознаёт медленнее реального времени (0.96x), энергия на короткий файл — 876 Дж против 82 Дж у giga-am. Соотношение качество/скорость/энергия только для русского языка не оправдывает использование.

### VAD
- Анализ Silero VAD не проводился, параметры не тюнились
- Периодически наблюдается «проглатывание» первых слогов фраз — с высокой вероятностью лечится снижением порога VAD или увеличением pre-speech padding. Отдельное исследование параметров Silero VAD запланировано на следующую итерацию, и результаты могут изменить общую картину качества.

### Ограничения методики
- Качество оценено субъективно на одном фрагменте
- Не тестировалась зашумлённая речь
- Не измерялась задержка (latency) — важная метрика для потокового режима
- - **Модели использовались с настройками по умолчанию** — параметры VAD (порог, min_silence_duration, min_speech_duration), размер окна, параметры декодирования ASR не оптимизировались. Результаты представляют baseline, а не предельные возможности
- Тесты на одном экземпляре оборудования, воспроизводимость на других конфигурациях не гарантируется
