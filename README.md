# Sherpa-ONNX Russian Lab
Сборник практических экспериментов по распознаванию (ASR), диаризации и синтезу (TTS) русской речи на базе библиотеки [sherpa-onnx](https://github.com/k2-fsa/sherpa-onnx) для Python.

> **Личный полигон для тестов**: код предоставляется "как есть", без идеальной архитектуры и поддержки.

## Цели проекта
- Сборка работающих примеров ASR (распознавание речи) для русского языка.
- Тестирование разных моделей ASR и вариантов диаризации (разделение по ролям).
- Тестирование разных настроек ASR, сегментации, эмбедингов и т.д.
- Создание скриптов для формирования протоколов совещаний.
- Голосовое взаимодействие с LLM-агентами (в перспективе).

## Особенности подхода
Проект строится на принципах максимальной легкости и минимального количества зависимостей:
- **Минимализм**: Полный отказ от тяжелых зависимостей вроде `torch`, `torchaudio` или `librosa`. 
- **Легкий стек**: Только `sherpa-onnx`, `numpy`, `pyyaml` и `ffmpeg`. 
- **Поточная обработка**: Чтение аудио напрямую из `stdout` FFmpeg через `subprocess` в `numpy`-буфер.
- **Скорость**: Инференс на ONNX Runtime позволяет работать с приемлемой производительностью на CPU офисных ноутбуков без дискретных видеокарт.
- **Простота**: Скрипты не требуют настройки сложных окружений и огромных объемов RAM.

## Технологический стек
- **Движок**: `sherpa-onnx` (ONNX Runtime)
	- Репозиторий: https://github.com/k2-fsa/sherpa-onnx
	- Документация: https://k2-fsa.github.io/sherpa/onnx/index.html
	- Описание API: https://pub.dev/documentation/sherpa_onnx/latest/sherpa_onnx/
- **Язык**: Python 3.10+
- **Обработка звука**: FFmpeg (через стандартный `subprocess`)
- **Математика**: `numpy`
- **Железо**: Обычные офисные ноутбуки и ПК с современными CPU. Наличие GPU не требуется.

## Используемые модели ONNX:
- **ASR (Речь в текст)**: 
	  - `nemo-ctc-punct-giga-am-v3-russian-int8` (высокая скорость для русского языка и отличная пунктуация)
	  - `qwen3-asr-0.6B-int8` (многоязычность)
- **VAD (Определение голосовой активности)**:
	- `silero_vad.onnx`
- **Speaker ID (Эмбеддинги)**:
	- `3dspeaker_speech_eres2net_base_sv_zh-cn_3dspeaker_16k.onnx`
- **Segmentation**:
	- `sherpa-onnx-pyannote-segmentation-3-0`

## Как запустить

Требования к системе:
- должен быть установлен Python 3.10+;
- в PATH должен быть доступен `ffmpeg` (используется для перекодирования аудио в 16 кГц моно PCM).

1. Клонирование репозитория
```bash
git clone https://github.com/akret00/sherpa-onnx-russian-lab.git
cd sherpa-onnx-russian-lab
```
2. Создание виртуального окружения и установка зависимостей
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

3. Скачивание моделей
Создайте структуру директорий для моделей:
```bash
mkdir -p models/asr models/vad
```

**VAD (определение голосовой активности):**
```bash
wget https://github.com/k2-fsa/sherpa-onnx/releases/download/asr-models/silero_vad.onnx -O models/vad/silero_vad.onnx
```

**ASR-модели:**
_Для русского языка (рекомендуемая, ~0.45 ГБ ОЗУ):_
```bash
# GigaAM v3 — быстрая, точная на русском, ~430 МБ ОЗУ
wget https://github.com/k2-fsa/sherpa-onnx/releases/download/asr-models/sherpa-onnx-nemo-ctc-punct-giga-am-v3-russian-2025-12-16.tar.bz2
tar -xf sherpa-onnx-nemo-ctc-punct-giga-am-v3-russian-2025-12-16.tar.bz2 -C models/asr/gigaam-v3-onnx/
rm sherpa-onnx-nemo-ctc-punct-giga-am-v3-russian-2025-12-16.tar.bz2
```
_Опционально (мультиязычная, медленнее, ~2.2 ГБ ОЗУ):_
```bash
# Qwen3 ASR 0.6B — мультиязычная, требует больше ресурсов
wget https://github.com/k2-fsa/sherpa-onnx/releases/download/asr-models/sherpa-onnx-qwen3-asr-0.6B-int8-2026-03-25.tar.bz2
tar -xf sherpa-onnx-qwen3-asr-0.6B-int8-2026-03-25.tar.bz2 -C models/asr/qwen3-asr-0.6b/
rm sherpa-onnx-qwen3-asr-0.6B-int8-2026-03-25.tar.bz2
```

> **Примечание:** модель Qwen3-ASR 1.7B в репозитории sherpa-onnx не выложена, скачивается отдельно. Инструкция по её получению не включена — используйте по своему усмотрению.

### 4. Настройка конфигурации
Скопируйте образец конфигурации и отредактируйте под свои пути:
```bash
cp config.yaml.sample config.yaml
```

В файле `config.yaml` нужно:
1. **Выбрать активную ASR-модель** — раскомментировать одну из строк в секции `work`:
```yaml 
    work:
      asr_model_name: "giga-am-v3"   # ← раскомментируйте нужную
      # asr_model_name: "qwen3-0.6b"
```
	Имя должно совпадать с ключом в секции `models → asr`.

2. **Проверить пути к файлам моделей** — если вы скачивали модели в другую директорию, поправьте пути в секции `models → asr → <имя_модели>`.

### 5. Запуск
**Распознавание аудиофайла:**
```bash
# Базовый запуск (1 поток)
python ./src/asr_vad.py --input audio_file_name.wav
# С указанием количества потоков (ускоряет работу на многоядерных CPU)
python ./src/asr_vad.py --input audio_file_name.wav --num-threads 4
```
Поддерживаемые форматы: любые, которые умеет читать ffmpeg (wav, mp3, m4a, ogg, flac и др.).

**Распознавание с микрофона (потоковый режим):**
```bash
python ./src/asr_vad.py --mic
```
Скажите что-нибудь в микрофон — распознанный текст будет появляться в консоли. Для выхода нажмите `Ctrl+C`.

### 6. Проверка работоспособности
Быстрый тест: создайте короткий WAV-файл с речью (или скачайте любой sample) и запустите:
```bash
python ./src/asr_vad.py --input test.wav --num-threads 1
```

Если в консоли появился распознанный текст с временными метками — всё работает.
